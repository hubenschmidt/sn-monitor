# Mock Interview Script

Read each question aloud. Pause between questions to let the candidate respond.

---

## Phase 1 — Easy: [Two Sum (LC #1)](https://leetcode.com/problems/two-sum/)

> You're given a list of numbers and a target number. Find the two numbers in the list that add up to the target, and tell me their positions. Each input has exactly one answer, and you can't use the same number twice. For example, if the list is 2, 7, 11, 15 and the target is 9, the answer is positions 0 and 1 — because 2 plus 7 equals 9.

> Follow-up: How fast is your solution, and how much memory does it use?

---

## Phase 2 — Medium: [LRU Cache (LC #146)](https://leetcode.com/problems/lru-cache/)

> Design a "Least Recently Used" cache — basically a storage system with a maximum size. It should support two operations: "get" looks up a value by its key and returns negative one if the key isn't there, and "put" adds or updates a key-value pair. When the cache is full, it should automatically remove the item that hasn't been used in the longest time. Both operations need to be as fast as possible.

> Follow-up: Walk me through what happens step by step when we do: put 1-1, put 2-2, get 1, then put 3-3 — and the cache can only hold 2 items.

---

## Phase 3 — Hard: [Trapping Rain Water (LC #42)](https://leetcode.com/problems/trapping-rain-water/)

> Imagine a bar chart where each bar has a different height. After it rains, water gets trapped between the taller bars. Given a list of bar heights, figure out how much total water is trapped. For example, with heights 0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1 — the answer is 6 units of water.

> Follow-up: Can you solve it using a "two-pointer" approach that uses very little extra memory?

---

## Final Review

> Pick whichever solution you think could be improved the most and give me a final optimized version. Walk me through what you changed and why.
